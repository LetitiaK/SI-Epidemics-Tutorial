1 The networks

Throughout this document, the following four example networks will be used:

1.1 Example Network I: Dolphins

The first example network is a social network of bottlenose dolphins. The nodes are the bottlenose dolphins (genus Tursiops) of a bottlenose dolphin community living off Doubtful Sound, a fjord in New Zealand (spelled fiord in New Zealand). An edge indicates a frequent association. The dolphins were observed between 1994 and 2001.

  • Type: Undirected, unweighted
  • Vertices (Dolphins): 62
  • Edges (Associations): 159
  • Category: Animal

Source: http://konect.uni-koblenz.de/networks/dolphins

1.1.1 Morbillivirus Infections

Dolphins, porpoises, and whales with clinical morbillivirus infection have exhibited the following symptoms:

  • Skin lesions
  • Pneumonia
  • Brain infections
  • Secondary or latent infections

Morbilliviruses are usually spread through inhalation of respiratory particles or direct contact between animals, including between mothers and calves. It cannot be transmitted to humans. Animals can also be exposed to the virus through other entryways such as the eyes, mouth, stomach, skin wounds, and the urogenital tract.
For more information, see, e.g., http://www.nmfs.noaa.gov/pr/health/mmume/midatlantic2013/morbillivirus_cetaceans.pdf

1.2 Example Network II: U.S. Airports

This dataset is the complete US airport network in 2010. The data is downloaded from the Bureau of Transportation Statistics (BTS) Transtats site. Each edge represents a connection from one airport to another, and the weight of an edge shows the number of seats available on that connection in the given direction.

  • Type: Directed, weighted
  • Vertices (Airports): 1574
  • Edges (Flights): 28235
  • Category: Infrastructure

Source: https://toreopsahl.com/datasets/#usairports

1.2.1 Infectious diseases

In case of the U.S. airport network, it is assumed that all passengers that arrive at an “infected” airport are immediately also infected and that all passengers that leave an “infected” airport are also infected. The epidemic parameters for an infectious disease are set as follows:

  • Beta: 0.09
  • Infected Hosts at beginning: 10

1.3 Example Network III: Gnutella

Gnutella is a large peer-to-peer network. In 2007 it was the most popular file sharing network on the Internet. This dataset is from August 31, 2002.

  • Type: Directed, unweighted
  • Vertices (Host): 62586
  • Edges (Connection): 147892
  • Category: Computer

Source: https://snap.stanford.edu/data/p2p-Gnutella31.html

1.3.1 Malware Infections

For the Gnutella network, any kind of malware infection, e.g. viruses or worms, might be applicable. In this case, however, we assume a very fast spreading malware. The epidemic parameters are therefore set as follows:

  • Beta: 0.4
  • Infected Hosts at beginning: 10

1.4 Example Network IV: Gowalla

Gowalla was a location-based social network launched in 2007, where users shared their location by checking in. It was acquired by Facebook on December 2, 2011 and shout down on March 10, 2012. For this dataset 6.442.890 check-ins were collected over a period of Feb. 2009 - Oct. 2010.

  • Type: Undirected, unweighted
  • Vertices (User): 196591
  • Edges (Friendship): 950327
  • Category: Social

Source: http://snap.stanford.edu/data/loc-gowalla.html

1.4.1 Infectious diseases

For the Gowalla network, it is assumed that all virtual friendships exist also in real life. Hence, the Gowalle network is used to model the spreading dynamics of an infectious disease. The epidemic parameters are set as follows:

  • Beta: 0.05
  • Infected Hosts at beginning: 10

2 SI Models

The SI-Model is also called the simple epidemic model. In this case, there are only two classes namely the susceptible class and the infected class. Since vital dynamics are not considered in this model, the contact number \(\sigma = \infty\) meaning that there is no epidemic threshold and eventually everyone is infected. The contact number (or average reproductive rate) is the average number of adequate contacts per infective per infectious period.

library(igraph) 
library(animation)
library(xtable)

# Set your working directory - this is especially important if you want to save the animation
# setwd("INSERT HERE YOUR WORKING DIRECTORY")
# Load a network from a .csv file
dat=read.csv(file.choose(),header=TRUE)
el=as.matrix(dat)
el[,1]=as.character(el[,1])
el[,2]=as.character(el[,2])
n=graph.edgelist(el,directed=FALSE)

2.1.3 Basic settings

# Set N as the number of individuals, i.e. vertices in the network
N <- vcount(n) 

# Set init_inf as the number of initially infected individuals
init_inf = 2

# Set simlength as the number of time steps - might need to be adjusted after first trial
# Therefore, see, e.g., the prevalence table created below to get a better overview
simlength <- 30

# Set simnumber as the number of simulations
simnumber <- 10

# Set beta
beta <- 0.20

# Switch to set whether you would like to plot the spreading of the epidemic over the network 
# Caution: might significantly slow down the simulation when TRUE
plot.spread <- TRUE

# Generate an edge list from the loaded network dataset.
# Caution: "names" MUST be set as FALSE!
# >> Otherwise, the vertex names are saved as strings, which will lead to an error 
links <- as_edgelist(n, names = FALSE)

# infdata_sum is a list, which will be used to save the list of infection data of each simulation round
infdata_sum = list()
# susdata_sum is a list, which will be used to save the list of susceptible data of each simulation round
susdata_sum = list()
# incidence_sum is a list, which will be used to save the list of the incidence data of each simulation round
incidence_sum = list()

2.1.4 Simulation

# The for loop is used to run the simulation as often as defined in simnumber
for (j in 1:simnumber){
 
# Initialization of the infection status - at the beginning all individuals are susceptible
infected <- logical(N) 

# Selecting the "patient zero". In this case the first infected individuals are picked randomly
# In this case there are initially two infected individuals.
# Example: If one would like to manually select the initially infected individual (vertex 5), 
# one can select:
# patientzero <- 5
patientzero <- sample(N,init_inf) 

# The initially selected infected individuals are now set to be infected
infected[patientzero] <- TRUE

# The number of susceptibles is hence the total number of individuals N - the number of infected individuals
# The number of infected individuals is found by sum(infected)
susceptible = N - sum(infected)

# infdata is a list, which is used to save the infection data of each timestep within one simulation round
# As described above the whole list of each simulation round is saved in infdata_sum 
# (i.e. infdata_sum is a nested list)
infdata = list(sum(infected))

# susdata is a list, which is used to save the susceptible data of each timestep within one simulation round
# As described above the whole list of each simulation round is saved in susdata_sum 
# (i.e. susdata_sum is a nested list)
susdata = list(susceptible)

# The incidence is the average number of new cases per day caused by all infectives.
# At timestep 0 the list is empty
incidence = list()

########################################################################################################

# Here the parameters for the animated simulation are set
if (plot.spread) {
  # Remember to adjust this, if the graph is directed
    network.i <- graph.edgelist(links, directed=FALSE)
    # Store a fixed layout for the graph
    fixlayout <- layout.kamada.kawai(network.i)  
    # Initialize the vertex colors (SkyBlue2 is also the default node colour in igraph)
    # As a first step all vertices are colored blue
    node.colour <- rep("SkyBlue2",N) 
    # Infected vertices will be coloured red
    node.colour[patientzero] <- "red" 
  # In order to save the network at timestep 0, it must be saved in this step, otherwise this step will
  # be missing in the simulation
  # The filename of this plot will be sim.000. This is important as described below!  
    png(paste("sim", 000, ".png", sep = "."))
  # Now the network is plotted. The heading of the plot shows the respective timestep  
    plot(network.i,layout=fixlayout, main="Time = 0", vertex.color=node.colour)
    # It is important to set dev.off after saving the image, because it will otherwise not be accessible
    dev.off()
}

########################################################################################################

# As described above it is important to save the first plot as *.000.
# This is, because the .png files of each timestep are later on set together to a single .gif file
# During the simulation, the filenames are created on the fly 
# (in this way they are automatically adjusted if the number of timesteps is changed).
# If they are saved without leading zeros, i.e. without padding, the convert function of ImageMagick
# will sort the files as 1,10,11,2,3,...
# In order to achieve the correct sorting, the filenames must be padded > 000, 001,002,003,...010,011,...
# Therefore, the following function is used
pad_int<-function(n,scale){
  out_string<-paste(10*scale + n,sep='')
  out_string<-substr(out_string,2,nchar(out_string))
  return(out_string)
}

########################################################################################################

# Here the actual simulation starts
for (i in 1:simlength) {
  # Find the indeces of edges that connect an infected individual to an uninfected
  discordant.links <- which(xor(infected[links[,1]],infected[links[,2]])) 
  # Determine randomly which of the discordant edges transmits the disease
  transmit <- rbinom(length(discordant.links),1,beta) 
  # The infection vector is updated
  transmitter.links <- discordant.links[transmit==1]
  # Gets both vertices of the transmitter links into a single vector; unique just filters out repetitions
  nodes.of.transmitter.links <- unique(as.vector(links[transmitter.links,1:2])) 
  # Both vertices are set to TRUE (although the transmitter already had 'TRUE'). 
  # In more complex models, you might want to overwrite only the newly infected vertices.
  infected[nodes.of.transmitter.links] <- TRUE 
  # As mentioned above the infdata list is used to save the number of infected individuals at each timestep
  # In order to save also the initial number, i.e. init_inf = 2, the current number must be saved at i+1
  infdata[[i+1]]= sum(infected)
  # The same explanation as with infdata applies also for susdata
  susdata[[i+1]]= N - sum(infected)
  # The incidence is calculated as the number of susceptibles at timestep i minus the number or susceptibles
  # at timestep i+1
  incidence[[i]]=susdata[[i]] - susdata[[i+1]]
  # If the animation is desired the following steps will be executed
  if (plot.spread) {
    # Each newly infected vertex will be colored in red
    node.colour[infected] <- "red"
    # Here the padding of the filename is adjusted.
    # In this case, we have a padding of 3 digits (>100), meaning 001,002,...010,...
    # If more than 100 files are ment to be saved, the padding would have to be adjusted accordingly to 1000
    padded<-pad_int(i,100)
    # The filename is created on the fly and padded - each filename contains the timestep i
    png(paste("sim",padded,".png", sep=""))
    # The acutal plotting of the network at each timestep
    plot(network.i,layout=fixlayout, main=paste("Time =", i), vertex.color=node.colour)
  }
  # As described above, dev.off() after creating the .png files is required
  # CAUTION!: If you set plot.spread to FALSE, you must either delete dev.off() or set it into comments
  dev.off()
}
# After the simulation is completed, the whole infdata list is saved within infdata_sum (nested list)
  infdata_sum[[j]] = infdata
# After the simulation is completed, the whole susdata list is saved within susdata_sum (nested list)
  susdata_sum[[j]] = susdata
# After the simulation is completed, the whole incidence list is saved within incidence_sum (nested list)
  incidence_sum[[j]] = incidence
}

In the following, this question will be answered for the vertices connected by the diameter path of the dolphin network.

# Since we will now start at exactly one vertex, init_inf must be changed to 1
init_inf = 1

# When searching for the average number of timesteps, the results might vary significantly for only
# 10 simulation rounds. Therefore a higher number, e.g., 100 simulation rounds are advisable
simnumber <- 100

# Then, use the farthest_vertices function in order to display the two vertices that are 
# connected by the diamter path. The function shows also the length of the path. In this case: 7.
# In this case, verices 1 and 50
diameter <- farthest_vertices(network.i)
print(diameter)
## $vertices
## + 2/62 vertices:
## [1]  3 50
## 
## $distance
## [1] 7
# OPTIONAL: For a better overview, the two vertices connected by the diameter path are highlighted in yellow
network.i=graph.edgelist(links,directed=FALSE)
fixlayout <- layout.kamada.kawai(network.i)
node.colour <- rep("SkyBlue2",N) 
node.colour[1] <- "yellow" 
node.colour[50] <- "yellow"
plot(network.i,layout=fixlayout, main="The vertices connected by the diameter path", vertex.color=node.colour)

# A new list spreadingSpeed_sum is required in order to save the timestep of each simulation round,
# at which the farthest vertex is reached
spreadingSpeed_sum = list()

######### SIMULATION #########

# Now the simulation is repeated, starting at vertex 1
for (j in 1:simnumber){
infected <- logical(N) 

# Initialization of the farinfected status
# This is used to track at which timestep, the farthest vertex is reached
# At timestep zero all elements are set to FALSE
farinfected <- logical(simlength)

# The first patient, patientzero, is now selected to be vertex 1
patientzero <- 1 
infected[patientzero] <- TRUE

for (i in 1:simlength) {
  discordant.links <- which(xor(infected[links[,1]],infected[links[,2]])) 
  transmit <- rbinom(length(discordant.links),1,beta) 
  transmitter.links <- discordant.links[transmit==1]
  nodes.of.transmitter.links <- unique(as.vector(links[transmitter.links,1:2])) 
  infected[nodes.of.transmitter.links] <- TRUE
  
  # Check if vertex 50 is infected. If yes, set farinfected[[i]] to TRUE
  if(infected[[50]] == TRUE) {
    farinfected[[i]] <- TRUE
  }
  
  # At the moment that farinfected[[i]] is set to TRUE for the first time, the sum of farinfected is
  # exactly 1. The timestep, at which this happens is then saved in spreadingSpeed
  if(sum(farinfected) == 1) {
    spreadingSpeed = i
  }
}
# After each simulation round the timestep saved in spreadingSpeed is added to spreadingSpeed_sum list
  spreadingSpeed_sum[[j]] = spreadingSpeed 
}

# After all simulation rounds have run, the average number of timesteps is calculated.
# First the sum of all timesteps is calculated
sum <- Reduce("+", spreadingSpeed_sum)
# The the sum is devided by the number of simulation rounds
avg <- sum/simnumber
# The result is printed
print(paste("On average vertex 50 is infected after", avg, "timesteps."))
## [1] "On average vertex 50 is infected after 9.54 timesteps."
# It might be also interesting to see the minimum number of timesteps that were required during

fastest <- min(unlist(spreadingSpeed_sum)) 
print(paste("In the fastest case, vertex 50 is infected after", fastest, "timesteps."))
## [1] "In the fastest case, vertex 50 is infected after 4 timesteps."
# Similarly, one can also check the maximum number of timesteps

## [1] "In the slowest case, vertex 50 is infected after 27 timesteps."
# In order to get a better overview, the spreadingSpeed_sum list is converted to a data frame

2.2 U.S. Airports

The U.S. Airport network is a weighted network. This means that the probability of infection is not only dependent on beta, but also on the weight of the edge. In addition, the network is a weakly-connected digraph meaning that vertices with in-degree 0 might never be infected, if they are not initially infected. If the initially infected vertices have out-degree 0, the epidemic will not spread at all. Note that in the following, parts that are equal to the ones above are not shown and that only those parts that need to be adjusted are commented. Furthermore, note that for this network the plotting will be set to FALSE, because of the large size of the network.

# Load a network from a .csv file
el=read.csv(file.choose(), header = TRUE) 
el=as.matrix(el)
el[,1]=as.character(el[,1])  
el[,2]=as.character(el[,2])
# Because the .csv file of a weighted graph contains three columns the edgelist must be defined for the
# first two columens only: el[,1:2]
# Because the network is a digraph, directed must be set to TRUE
n=graph.edgelist(el[,1:2], directed = TRUE) 
# The weight of all edges is saved as attribute of the edges.
# The weight is stored in the third column
E(n)$weight=as.numeric(el[,3]) 
for (j in 1:simnumber){
  infected <- logical(N) 
  patientzero <- sample(N,init_inf)  
  infected[patientzero] <- TRUE
  susceptible = N - sum(infected)
  infdata = list(sum(infected))
  susdata = list(susceptible)
  incidence = list()
  time = list(0)

    if (plot.spread) {
      network.i=graph.edgelist(links, directed=TRUE)
        fixlayout = layout.kamada.kawai(network.i)  
        node.colour = rep("SkyBlue2",N) 
      node.colour[patientzero] <- "red"
      png(paste("sim", 000, ".png", sep = "."))
        plot(network.i,layout=fixlayout, main="Time = 0", vertex.color=node.colour)
        dev.off()
    }

  pad_int = function(n,scale){
    out_string = paste(10*scale + n,sep='')
    out_string = substr(out_string,2,nchar(out_string))
    return(out_string)
  }

    for (i in 1:simlength) {
      # k is defined as a counter variable in order to loop through all discordant edges
      k = 1
      # Find the indeces of edges that connect an infected individual to an uninfected
      # In case of directed networks this must be adjusted, because the epidemic can spread only from
      # the edge tail to the edge head. Therefore, only does edges that connect an infected 
      # individual (at the tail) with an uninfected individual (at the head) are required
      discordant.links <- which(infected[links[,1]])
        # The following code is only executed if there is at least one discordant edge
        if(length(discordant.links) >= 1) {
          
          # If there is more than one discordant edge the code must be executed for all discordant edges
          for (m in 1:length(discordant.links)) {
            # The counter variable k is used to select the respective discordant edge
            selectedEdge = discordant.links[k]
            # For each selected discordant edge the corresponding edge weight is selected
            edgeWeight = E(n)$weight[selectedEdge]
            # Another counter variable x is defined 
            x = 0
            
            # The following code is repeated until one of breaking criteria applies
            # This is because the weight of the edge determines how often, i.e. by how many passengers,
            # the disease can be possibly transmitted during one timestep
            # Note, however, that the disease is not transmitted more than ones, i.e. if the disease is
            # transmitted the repeating loop stops
            repeat{
              # The rbinom function is used to determine the transmission for each edge
              transmitDisease = rbinom(1,1,p.t)
                # If the disease is transmitted, i.e., trasmitDisease == 1,
                # the next discordant edge is selected
                if(transmitDisease == 1){
                  transmitter.links <- discordant.links[k]
                  nodes.of.transmitter.links <- unique(as.vector(links[transmitter.links,1:2])) 
                  infected[nodes.of.transmitter.links] <- TRUE 
                  # In order to select the next discordant edge, the counter variable k is increased
                  k = k + 1
                  break
                }
              # If the disease is not transmitted the counter variable x is increased
              # in order to count how often the repeat loop has been repeated
              x = x + 1
                # If the repeat loop has been repeated as often as the weight of the edge,
                # this implies that the disease has not been transmitted during this timestep
                # Hence, this is the second breaking criterion
                if (x == edgeWeight) {
                  # In order to select the next discordant edge, the counter variable k is increased
                  k = k + 1
                  break
                } 
            } 
          }
        } 
     
      infdata[[i+1]]= sum(infected)
      susdata[[i+1]]= N - sum(infected)
      incidence[[i]]=susdata[[i]] - susdata[[i+1]]
          
        if (plot.spread) {
            node.colour[infected] <- "red"
            print("THE COLOR")
            print(node.colour)
            padded<-pad_int(i,100)
            png(paste("sim",padded,".png", sep=""))
            plot(network.i,layout=fixlayout, main=paste("Time =", i), vertex.color=node.colour)
        }
      
      #dev.off()
    }
    
  infdata_sum[[j]] = infdata
  susdata_sum[[j]] = susdata
  incidence_sum[[j]] = incidence
}

2.3 Gnutella

The Gnutella network is a weakly-connected digraph. This means that depending on the initially infected vertices and existing edges, the epidemic might spread only between very few vertices. Also, vertices with in-degree 0 might never be infected, if they are not initially infected. If the initially infected vertices have out-degree 0, the epidemic will not spread at all. Note that in the following, parts that are equal to the ones above are not shown and that only those parts that need to be adjusted are commented.

# Load a network from a .csv file and change directed to TRUE
dat=read.csv(file.choose(),header=TRUE)
el=as.matrix(dat)
el[,1]=as.character(el[,1])
el[,2]=as.character(el[,2])

2.3.1 Analysis

2.3.1.1 What is the percentage of infected vertices when the epidemic starts at the vertex with the highest out-degree?

In the epidemic context, a super-spreader is a vertex (human, animal, computer etc.) that is alone responsible for a very high number of new infections. In the following, this will be shown exemplarily for the Gnutella network.

# First the vertex with the highest outdegree needs to be found
maxdeg <- which(degree(n, mode = "out")==max(degree(n, mode = "out")))
max <- max(degree(n, mode = "out"))
print(paste("The vertex with the highest out-degree, is vertex number", maxdeg, "with an out-degree of", max,"."))
## [1] "The vertex with the highest out-degree, is vertex number 8950 with an out-degree of 78 ."
# Because the epidemic starts now only at one vertex, init_inf must be set to 1
init_inf = 1

# In this case the epidemic will spread faster and it is sufficient to view 50 timesteps
simlength <- 50

# A new list infdata_end is created in order to save final number of infected vertices after each simulation
# round
infdata_end = list()

for (j in 1:simnumber){
infected <- logical(N) 

# patientzero is now set to be the vertex with the highest out-degree
patientzero <- maxdeg

infected[patientzero] <- TRUE
susceptible = N - sum(infected)
infdata = list(sum(infected))
susdata = list(susceptible)
incidence = list()


for (i in 1:simlength) {
  discordant.links <- which(infected[links[,1]]) 
  transmit <- rbinom(length(discordant.links),1,beta) 
  transmitter.links <- discordant.links[transmit==1]
  nodes.of.transmitter.links <- unique(as.vector(links[transmitter.links,1:2])) 
  infected[nodes.of.transmitter.links] <- TRUE
